# BUG-005: ドラッグ&ドロップによる移動が全く動作しない

- **State**: Open
- **Reported**: 2026-07-12
- **Area**: file-explorer/tree-view（DnD）/ Tauri window 設定

## Summary

ノート・フォルダのドラッグ&ドロップ移動が実機（実マウス操作）でまったく動作しない。

## Reproduction

1. 保管庫を開く
2. ノートまたはフォルダを別フォルダへドラッグ&ドロップ
3. 観測結果: 何も起きない（移動されない・ドロップ先ハイライトも出ない）

## Expected

HTML5 ドラッグ&ドロップでノート/フォルダが移動でき、ドロップ先がハイライトされる。

## Cause

Tauri v2 のウィンドウは既定で `dragDropEnabled: true`。この状態では OS レベルの
ファイルドラッグ&ドロップハンドラが WebView を覆い、**WebView 内の HTML5 DnD イベント
（dragstart/dragover/drop）が発火しない**。フロントの DnD 配線・`move_path` 自体は正しいが、
イベントが届かないため一切動作しなかった。

## Fix

`src-tauri/tauri.conf.json` のウィンドウ設定に `"dragDropEnabled": false` を追加し、
OS のファイルドロップ処理を無効化して WebView の HTML5 DnD を有効化する。

- **Fix commit**: {hash}

## Notes

Linux headless では HTML5 DnD の自動発火が難しいため、Windows 実機での再確認が必要。
`move_path` の移動先計算・自己/子孫拒否は Rust ユニットテストで担保済み。

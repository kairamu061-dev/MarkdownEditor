# BUG-008: 引用文（`>`）内での Backspace / 入力が誤動作する

- **State**: Fixed
- **Reported**: 2026-07-16
- **Area**: editor/core（@codemirror/lang-markdown）

## Summary

`> 引用文` の行で Backspace を押すと、カーソル行より 1 行上の文字が消える、または入力内容が重複して反映されるなど、カーソル・編集操作が正常に動作しない。

## Reproduction

1. `> 引用テキスト` を含む文書を開く
2. 引用行内でカーソルを行中央に置く
3. Backspace を押す

観察: カーソル行ではなく 1 行上の文字が削除される、または入力文字が複数回挿入される。

## Expected

引用行内の Backspace はカーソル直前の 1 文字を削除する（通常の編集動作）。

## Cause

`@codemirror/lang-markdown` 6.5.0 の `deleteMarkupBackward` コマンドにバグがある。
`Blockquote` ノードに対する分岐条件が `!inner.item`（item が null のとき常に true）で常に enter していたため、削除位置の計算が誤った行になった。

旧コード（6.5.0）:
```js
(!inner.item || line.from <= inner.item.from || !/\S/.test(line.text.slice(0, inner.to)))
```

新コード（6.5.1）:
```js
((inner.item && line.from <= inner.item.from) || /^[\s>]*$/.test(line.text.slice(0, inner.to)))
```

## Fix

`@codemirror/lang-markdown` を 6.5.0 → 6.5.1 に更新。

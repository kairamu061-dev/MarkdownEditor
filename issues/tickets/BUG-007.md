# BUG-007: サイドバー開閉トグル（☰ ボタン）が効かない

- **State**: Fixed
- **Reported**: 2026-07-12
- **Area**: app-shell（src/shell/sidebar.ts, index.html）

## Summary

タイトルバーの ☰ ボタンをクリックしてもサイドバーが開閉しない。
フレームレス化（`4260d13`）以降のリグレッション。

## Reproduction

1. アプリを起動する
2. タイトルバー左端の ☰ ボタンをクリックする
3. サイドバーが開閉しない（何も起きない）

## Expected

☰ クリックでサイドバーが非表示⇔表示にトグルする（Ctrl+B と同じ挙動）。

## Cause

フレームレス化（`4260d13`）で `#sidebar-toggle`（☰）を `#app` 内から新設の `#titlebar` へ移動したが、
`initSidebar(root)` は `root = #app` に対して `root.querySelector("#sidebar-toggle")` でボタンを
探したままだった。ボタンが見つからず（null）、クリックリスナーが一度も配線されていなかった。
Ctrl+B は `window` の keydown 監視のため影響を受けない。

## Fix

`src/shell/sidebar.ts` でボタンの取得を `document.getElementById("sidebar-toggle")` に変更し、
配置に依存しないようにした。`initSidebar` の `root` 引数は不要になったため削除し、
`main.ts` の呼び出しも合わせて更新。

## Notes

`4260d13` の実機確認（2026-07-12）ではタイトルバーのウィンドウ操作は確認済みだったが、
☰ の開閉までは確認項目に含まれておらず見逃した。
修正後、Xvfb 上のリリースバイナリで ☰ クリック → 非表示 → 再クリック → 表示、Ctrl+B でも同様に
開閉することをスクリーンショットで確認済み（app-shell E-02/E-03 をフレームレス版で再実施）。

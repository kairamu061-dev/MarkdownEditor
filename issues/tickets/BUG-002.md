# BUG-002: ノートを開くと畳んでいたフォルダがすべて開いてしまう

- **State**: Open
- **Reported**: 2026-07-08
- **Area**: file-explorer/tree-view（`src/explorer/index.ts`）

## Summary

フォルダビューでフォルダを畳んだ状態にしても、いずれかのノートを開くと畳んでいたフォルダがすべて展開されてしまう。

## Reproduction

1. 複数フォルダを含む保管庫を開く
2. いくつかのフォルダを畳む（クリックで折りたたむ）
3. 任意のノートをクリックして開く
4. 観測結果: 畳んでいたフォルダがすべて展開状態に戻る

## Expected

ノートを開いても、各フォルダの開閉状態は維持される。

## Cause

折りたたみ状態が DOM 上の `li.collapsed` クラスとしてのみ保持されていた。
`openNote()` が毎回 `renderTree()` を呼び、`renderNodes()` がツリー全体を作り直すため、
`collapsed` クラスの付いていない新しい `<li>` に置き換わり、状態が失われていた。

## Fix

折りたたみ状態を DOM ではなく JS 側の状態（`ExplorerState.collapsedDirs: Set<string>`、
フォルダの相対パスを保持）で管理するように変更。`renderNodes()` は描画時に
このセットを参照して初期の開閉状態とアイコンを決め、フォルダのクリックでセットを更新する。
これにより再描画をまたいで開閉状態が保持される。

- **Fix commit**: 445ec66

## Notes

この状態管理は後続の tree-view 拡張（開閉マーク・DnD 等）でも土台として利用する。

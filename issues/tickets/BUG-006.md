# BUG-006: redo（Ctrl+Y）が動作しない

- **State**: Closed
- **Reported**: 2026-07-12
- **Area**: editor/core（キーマップ）

## Summary

undo（Ctrl+Z）は動作するが、redo（Ctrl+Y）が動作しない。

## Reproduction

1. エディタで文字を入力
2. Ctrl+Z で undo（→ 動作する）
3. Ctrl+Y で redo
4. 観測結果: 何も起きない

## Expected

Ctrl+Y（および Windows で一般的な Ctrl+Shift+Z）で redo できる。

## Cause

CodeMirror の `historyKeymap` は Windows 向けには **redo を `Ctrl+Y` のみ**に割り当てており
（`Ctrl+Shift+Z` は Linux 限定バインド）、`Ctrl+Y` が WebView 側で拾われた場合に
フォールバックがない。実機で `Ctrl+Y` が redo に届かず、代替の `Ctrl+Shift+Z` も未割当だった。

## Fix

`src/editor/index.ts` のキーマップに、`redo` を **`Ctrl+Y` と `Ctrl+Shift+Z` の両方**へ
明示バインドした keymap を高優先度（`Prec.highest`）で追加する。これにより Windows で一般的な
両方の redo 操作が確実に効く。

- **Fix commit**: 1803e76

## Notes

Windows 実機での Ctrl+Y / Ctrl+Shift+Z 双方の確認が必要。
2026-07-12 にユーザーが Windows 11 実機で redo を確認し問題なし → Closed。

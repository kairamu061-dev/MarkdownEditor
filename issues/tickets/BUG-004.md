# BUG-004: ツリー空白部の右クリックで WebView2 既定メニューが出る / ルート作成・ドロップの当たり判定が狭い

- **State**: Fixed
- **Reported**: 2026-07-09
- **Area**: file-explorer/tree-view（src/explorer/explorer.css, index.ts）

## Summary

サイドバー下方（ツリー内容より下の余白）を右クリックすると、アプリのコンテキストメニューではなく
WebView2/Edge の既定メニュー（戻る・印刷…）が表示される。ルート向けメニューとルートへのドロップは
ツリー内容直下の狭い帯でしか反応しない。

## Reproduction

1. ノート数の少ない保管庫を開く
2. サイドバーのツリーより下の余白を右クリック
3. WebView2 の既定コンテキストメニューが表示される（新規ノート/新規フォルダのメニューが出ない）

## Expected

サイドバー内の余白はどこを右クリックしてもルート向けメニュー（新規ノート・新規フォルダ）が出る。
既定の WebView2 メニューはサイドバー内では表示されない。ルートへのドロップも同じ範囲で受け付ける。

## Cause

- `.tree` が `min-height: 80px` のみで、サイドバーの残り高さまで広がっていなかった。
  `.tree` 要素の外側の余白は contextmenu / drop ハンドラの対象外
- サイドバー内で WebView2 の既定コンテキストメニューを抑止していなかった

## Fix

- `.tree` を `min-height: 100%` にしてサイドバーコンテンツの余白全体を覆うようにした
  （ルート向け右クリック・ルートへのドロップが余白全体で反応する）
- サイドバー全体（#sidebar）で `contextmenu` の既定動作を抑止（アプリのメニュー表示には影響しない）

- **Fix commit**: （このチケットと同一コミット）

## Notes

2026-07-09 の Windows 実機検証（tmp/フィードバック/動作確認結果.md FINDING-2）で発見。
エディタ側（メイン領域）の既定メニュー抑止は本チケットの範囲外（必要なら別途検討）。
サイドバー内の入力欄（リネーム等）でも既定メニューが出なくなるが、キーボード操作（Ctrl+C/V）は可能。

# BUG-009: 箇条書きの深いネストが視覚的に判別しにくい / Tab 連打後 Enter で最上位に戻る

- **State**: Closed
- **Reported**: 2026-07-16
- **Area**: editor/live-preview

## Summary

`- ` 箇条書きのネストが深さ 3 段を超えると使いにくい。原因は 2 つ。
1. Tab 連打 + Enter をすると最上位レベルに戻る（空ネスト項目の Enter 挙動）
2. 行折り返し時にハンギングインデントがなく、深い段では視覚的にネスト差が分かりにくい

## Reproduction

1. `- l1` を入力して Enter
2. Tab を 3 回押す（`      - ` になる）
3. Enter を押す → `- `（最上位）に戻ってしまう（期待: `      - ` の継続）

## Expected

- Tab を押した段数ぶん深いネストで Enter が継続する（Obsidian 互換）
- 折り返し行がテキスト開始位置に揃うハンギングインデントで表示される

## Cause

**Tab 連打の挙動**: `indentMore`（2 スペース追加）を複数回押しても、中間レベルに内容がない場合、パーサーは最深段を「親リスト直下の 1 レベル下」としか認識しない。空のネスト項目で Enter すると `insertNewlineContinueMarkup` が「最初の空項目 = 1 段上へ」ルールを適用し、最上位まで戻る。

正しい入力手順は「Tab → 内容入力 → Enter」の繰り返し。この手順では任意の段数が使える。

**ハンギングインデント**: `live-preview.ts` が `• ` 幅のぶん行全体をずらす CSS を設定していないため、行折り返し時は左端に戻る。

## Fix

1. `live-preview.ts` の baseTheme に `:has(.cm-list-bullet)` ルールを追加してハンギングインデント（`padding-left / text-indent`）を付与。段数に依存しない固定オフセットで、折り返し行をテキスト開始位置に揃える。
2. Tab 連打後の Enter 挙動は `@codemirror/lang-markdown` の仕様範囲内。正しい操作手順をユーザに案内する（ドキュメント追記）。

- **Fix commit**: `9ccc41a`

## Notes

コード上の制限は 3 段ではない（`syntaxTree.iterate` にも `lezer` パーサにも深さ上限なし）。正しい手順（Tab → 入力 → Enter）で 10 段以上のネストが可能。
2026-07-20 の Windows 11 実機検証（`9ccc41a`）で、折り返し続き行がテキスト開始位置に揃うハンギングインデントを確認し Closed とした（深いネストの Tab タイプ手順は未実施だが仕様範囲内）。

# BUG-001: add-feature.sh が CRLF 改行のため Linux で実行不能

- **State**: Fixed
- **Reported**: 2026-07-04
- **Area**: .claude/skills/add-feature/add-feature.sh

## Summary

`/add-feature` スキルのシェルスクリプトが CRLF 改行で保存されており、Linux devcontainer 上で bash が `$'\r': command not found` を出して実行に失敗する。

## Reproduction

1. Linux devcontainer 上で `bash .claude/skills/add-feature/add-feature.sh app-shell` を実行
2. `line 6: $'\r': command not found` などのエラーで exit code 2

## Expected

フィーチャーエリアのディレクトリとテンプレートが作成される。

## Cause

スクリプトが Windows 側で CRLF 改行のまま作成され、git 管理下に入る前だったため `.gitattributes` の `* text=auto eol=lf` による正規化が効いていなかった。

## Fix

`sed -i 's/\r$//'` で LF に変換。以後は `.gitattributes` の設定によりリポジトリ内では LF が維持される。

- **Fix commit**: （このチケットと同一コミット）

## Notes

他の CRLF ファイル（*.md）は bash 実行されないため実害なし。checkout 時も `eol=lf` により LF が維持される。

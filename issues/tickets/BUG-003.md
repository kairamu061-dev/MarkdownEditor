# BUG-003: 保管庫スイッチャの一覧がフォルダ名でなくフルパス表示

- **State**: Fixed
- **Reported**: 2026-07-09
- **Area**: file-explorer/vault-switch（src/explorer/index.ts）

## Summary

保管庫スイッチャのプルダウン項目が `D:\...\av-vault` のようにフルパスで表示される。
仕様（vault-switch/spec.md 機能一覧 No.2）は「フォルダ名（basename）、ツールチップにフルパス」。

## Reproduction

1. Windows 11 で複数の保管庫を開いて recentVaults に履歴を作る
2. サイドバーの保管庫名をクリックしてスイッチャを開く
3. 一覧項目がフルパスで表示される（ヘッダの保管庫名は正しく basename）

## Expected

一覧項目はフォルダ名（basename）で表示され、フルパスはツールチップに出る。

## Cause

`src/explorer/index.ts` の `basename()` が `/` のみで分割していた。Windows の保管庫パスは
`\` 区切りのため分割されず、パス全体がラベルに表示された
（ヘッダ名は Rust 側 `root.file_name()` 由来のため正常）。

## Fix

`basename()` を `/` と `\` の両方で分割するよう修正（末尾区切りにも耐性を持たせた）。
保管庫相対パス（`/` 区切り）に対する既存の挙動は不変。

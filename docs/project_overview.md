# プロジェクト概要

## 目的・背景

Windows 11 上で動作する、軽量・省メモリ指向のマークダウンエディタを開発する。

既存の高機能マークダウンエディタ（Obsidian など）は Electron ベースであり、
アイドル時でも数百 MB のメモリを消費し起動も遅い。
本プロジェクトは Obsidian の編集体験（ライブプレビュー、ファイルツリー、キーボード中心の操作）を参考にしつつ、
動作の軽さとメモリ使用量の節約を最優先の設計目標とする。

参考プロダクト: [Obsidian](https://obsidian.md/)
参考カラースキーマ: [ObsidianNord](https://github.com/insanum/obsidian_nord)（[Nord](https://www.nordtheme.com/) ベース）

## スコープ

### 作るもの

- ローカルフォルダ（ヴォールト）内の Markdown ファイルの閲覧・編集
- CodeMirror 6 によるライブプレビュー編集（Obsidian ライク）
- ファイルツリー（サイドバー）によるファイルの閲覧・作成・リネーム・削除
- Nord カラースキーマによるダークテーマ UI
- Windows 11 向けデスクトップアプリとしての配布（MSI / NSIS インストーラ）

### 作らないもの

- クラウド同期・アカウント機能
- プラグインシステム（将来検討）
- モバイル対応
- Markdown 以外のファイル形式の編集
- リアルタイム共同編集

## 技術スタック概要

| 技術 | 用途 | 選定理由 |
|------|------|----------|
| Tauri 2 | アプリフレームワーク | WebView2 利用により Electron 比で大幅に省メモリ・小サイズ。軽さ最優先の要件に合致 |
| Rust | バックエンド（ファイル I/O、検索） | Tauri のネイティブ側。重い処理をフロントから分離し軽快さを維持 |
| TypeScript + Vite | フロントエンド | 型安全と高速な開発サイクル |
| CodeMirror 6 | エディタエンジン | Obsidian 本体と同じエンジン。ライブプレビュー編集の再現性が高い |
| Nord / ObsidianNord | カラースキーマ | CSS ベースでありほぼ直訳で移植可能 |

フレームワーク選定の比較検討（Electron / WinUI 3 / Qt との比較）は 2026-07-04 に実施し、
軽さ・省メモリと Obsidian ライクな編集体験の両立を理由に Tauri 2 + CodeMirror 6 を採用した。

## 全体アーキテクチャ

```
┌─────────────────────────────────────────┐
│ Tauri アプリ（単一ウィンドウ）           │
│ ┌─────────────────┐ ┌─────────────────┐ │
│ │ WebView2         │ │ Rust コア        │ │
│ │ (フロントエンド) │ │ (バックエンド)   │ │
│ │                  │ │                  │ │
│ │ - UI レイアウト  │←→│ - ファイル I/O  │ │
│ │ - CodeMirror 6   │IPC│ - ヴォールト管理│ │
│ │ - テーマ (CSS)   │ │ - 全文検索       │ │
│ └─────────────────┘ └─────────────────┘ │
└─────────────────────────────────────────┘
```

- フロントエンドは UI 描画と編集操作に専念し、ファイルアクセスはすべて Rust 側の Tauri コマンド経由（IPC）で行う
- フレームワークレスの軽量構成を基本とし、重量級 UI ライブラリ（React 等）は採用しない（省メモリ方針）

## 制約

- 対象 OS は Windows 11（WebView2 標準搭載を前提とする）
- 開発環境は Linux devcontainer のため、Windows ネイティブ動作の最終確認はユーザーの実機で行う
- メモリ使用量の目標: アイドル時 100MB 以下
- 起動時間の目標: 1 秒以内（ウォームスタート）

## UI デザイン方針

### コンセプト

Obsidian のダークテーマを踏襲した、落ち着いた寒色系（Nord）のミニマル UI。
装飾を排し、テキスト編集への没入を最優先する。

### カラーパレット

Nord カラースキーマ（ObsidianNord 準拠）を使用する。

| 用途 | カラーコード | 説明 |
|------|-------------|------|
| プライマリ（アクセント） | `#88C0D0` (nord8) | リンク、選択中アイテム、カーソル |
| セカンダリ | `#81A1C1` (nord9) | 見出し、サブアクセント |
| 背景（メイン） | `#2E3440` (nord0) | エディタ背景 |
| 背景（サイドバー） | `#3B4252` (nord1) | サイドバー・パネル背景 |
| 背景（選択・ホバー） | `#434C5E` (nord2) | 選択行、ホバー状態 |
| ボーダー・非活性 | `#4C566A` (nord3) | 区切り線、コメント、非活性テキスト |
| テキスト | `#D8DEE9` (nord4) | 本文テキスト |
| テキスト（強調） | `#ECEFF4` (nord6) | 強調テキスト、アクティブ項目 |
| エラー | `#BF616A` (nord11) | エラー表示 |
| 警告 | `#EBCB8B` (nord13) | 警告表示 |
| 成功 | `#A3BE8C` (nord14) | 成功表示 |

### タイポグラフィ

- UI: システムフォント（`Segoe UI Variable`, `Yu Gothic UI` フォールバック）
- エディタ本文: ユーザー設定可能。デフォルトはシステムフォント
- コードブロック: 等幅（`Cascadia Code`, `Consolas` フォールバック）

### コンポーネント方針

- 自前の軽量 CSS（カスタムプロパティでパレット定義）。CSS フレームワークは使わない
- コンポーネントはフラットデザイン、角丸 4px、影は最小限
- すべての主要操作にキーボードショートカットを用意する

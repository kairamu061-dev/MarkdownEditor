# settings/vault-restore 開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| フォルダ消失時に lastVault を消さない | 外付けドライブやネットワークドライブが一時的に外れているだけの場合、戻れば次回起動で復元される方が親切 |
| 引数 / MDE_VAULT で開いた場合も lastVault を更新 | 「最後に開いたものが次も開く」という一貫した心理モデルを保つ |
| 保存失敗はログのみ（オープンをブロックしない） | 記憶は利便性機能であり、本体機能（ヴォールトを開く）を止める理由にならない |

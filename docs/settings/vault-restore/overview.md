# settings/vault-restore 概要

## 目的・背景

前回開いた保管庫を記憶し、次回起動時に自動で開く。毎回フォルダを選び直す手間をなくす、
settings エリアの中で日常利用への効果が最も大きい機能。

## スコープ

### 作るもの

- 保管庫を開いた時点（pick_vault / initial_vault 成功時）で lastVault を保存
- 起動時の解決順: 起動引数 > 環境変数 MDE_VAULT > 記憶した lastVault
- 記憶したパスが存在しない場合のフォールバック（未オープン状態で起動）

### 作らないもの

- 複数保管庫の履歴・切り替え UI
- 保管庫を「閉じる」操作（記憶の明示的クリア）

## 制約

- vault コマンド（file-explorer/vault）の内部に統合し、フロントエンドの変更を伴わない
- lastVault の保存失敗は起動・オープンをブロックしない（ログのみ）

## 完了条件

- 保管庫を開いて終了 → 引数なしで再起動 → 同じ保管庫が自動で開く
- 記憶したフォルダを削除した状態で起動 → 「保管庫を開く」ボタンが表示される

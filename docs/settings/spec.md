# settings 仕様

## 機能一覧

| # | 機能名 | 説明 | サブ項目 |
|---|--------|------|---------|
| 1 | 設定の永続化 | settings.json の読み書きコマンド。壊れていてもデフォルトで継続 | store |
| 2 | ヴォールトの記憶 | ヴォールトを開いたら記録。起動時に 引数 > MDE_VAULT > 記憶 の優先順で自動復元 | vault-restore |
| 3 | 設定モーダル | ⚙ ボタンで開く。フォントサイズ（10〜32）・フォントファミリー（空欄=システム） | ui |
| 4 | フォント反映 | 保存時にエディタへ即反映、起動時に適用 | ui |

## 画面・操作フロー

1. ヴォールトを開いて作業し、アプリを終了する
2. 次回起動時、同じヴォールトが自動で開かれる（記憶したパスが存在しない場合は未オープン状態）
3. サイドバーヘッダの ⚙ を押すと設定モーダルが開く
4. フォントサイズ・フォントファミリーを変更して「保存」→ エディタに即反映され、以後の起動でも維持される
5. 「キャンセル」または Esc、モーダル外クリックで変更を破棄して閉じる

## 各画面 / 状態の詳細

### 設定モーダル

- **表示内容**: 半透明オーバーレイ上の中央ダイアログ。項目: フォントサイズ（number、10〜32）、フォントファミリー（text、空欄はシステムフォント）。ボタン: 保存 / キャンセル
- **操作**: 入力 → 保存で確定。Esc / キャンセル / オーバーレイクリックで破棄
- **遷移先**: 閉じて通常画面へ

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| settings.json が存在しない / 壊れている | デフォルト設定で起動（エラー表示なし）。次回保存時に正常なファイルを書き直す |
| 記憶したヴォールトのフォルダが存在しない | 未オープン状態で起動（「ヴォールトを開く」ボタン表示） |
| 設定の保存失敗 | ステータス表示「設定の保存に失敗しました」。UI 上の値は維持 |
| フォントサイズが範囲外 | 入力欄の min/max で制限（保存不可にはしない。clamp して保存） |

## 未対応ケース

- 存在しないフォント名の検証（ブラウザのフォールバックに任せる）
- 複数ヴォールトの履歴（記憶は直近 1 件のみ）

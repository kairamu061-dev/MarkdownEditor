# settings 概要

## 目的・背景

アプリの設定を永続化し、日常利用の体験を整える。
最重要はヴォールトの記憶（前回開いたフォルダを次回起動時に自動で開く）で、
毎回フォルダを選び直す手間をなくす。あわせてエディタのフォント設定を提供する。

## スコープ

### 作るもの

- 設定ファイル（JSON）の読み書き基盤（Rust 側、アプリ設定ディレクトリに保存）
- ヴォールトの記憶と起動時の自動復元
- 設定モーダル UI（エディタのフォントサイズ・フォントファミリー）

### 作らないもの

- テーマ切り替え（Nord ダーク固定のまま）
- キーバインドのカスタマイズ
- 設定の同期・インポート/エクスポート
- ホットキーや自動保存間隔などの詳細チューニング項目（必要になったら追加）

## 制約

- 設定ファイルの読み書きも Rust 側コマンド経由（フロントに fs を持たせない方針を維持）
- 保存先は Tauri の app_config_dir（Windows: `%APPDATA%\dev.kairamu061.markdowneditor`）
- 設定ファイルが壊れていても起動をブロックしない（デフォルト値で継続）

## 完了条件

- ヴォールトを開いて再起動すると、同じヴォールトが自動で開く
- 設定モーダルでフォントを変更するとエディタに即反映され、再起動後も維持される
- サブ項目（store / vault-restore / ui）の完了条件をすべて満たす

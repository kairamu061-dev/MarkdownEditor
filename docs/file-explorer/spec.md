# file-explorer 仕様

## 機能一覧

| # | 機能名 | 説明 | サブ項目 |
|---|--------|------|---------|
| 1 | 保管庫を開く | フォルダ選択ダイアログ、または起動引数 / 環境変数 `MDE_VAULT` で指定 | vault |
| 2 | ファイル I/O コマンド | ツリー取得・ノート読み書き（保管庫内パスのみ許可） | vault |
| 3 | ファイルツリー表示 | サイドバーに .md とフォルダの階層を表示。展開/折りたたみ | tree-view |
| 4 | ノートを開く・保存 | クリックでエディタに読み込み。変更は 700ms デバウンスで自動保存、Ctrl+S で即時保存 | tree-view |
| 5 | ノート新規作成 | サイドバーヘッダの [+] ボタン。保管庫直下に「無題のノート N.md」を作成し開く | file-ops |
| 6 | リネーム・削除 | ツリー項目の右クリックメニュー。削除は OS のごみ箱へ | file-ops |

## 画面・操作フロー

1. 起動直後（保管庫未指定時）はサイドバーに「保管庫を開く」ボタンが表示される
2. ボタンを押しフォルダを選択すると、ファイルツリーが表示される
3. ノートをクリックすると内容がエディタに表示され、ツリー上で強調される
4. 編集すると自動保存される（停止後 700ms）。Ctrl+S で即時保存もできる
5. [+] で新規ノートが作られ、そのままエディタで開かれる
6. 右クリックメニューからリネーム・削除ができる

## 各画面 / 状態の詳細

### サイドバー（保管庫未オープン）

- **表示内容**: 「保管庫を開く」ボタンのみ
- **操作**: ボタン押下 → フォルダ選択ダイアログ
- **遷移先**: ファイルツリー表示

### サイドバー（ファイルツリー）

- **表示内容**: 保管庫名（ヘッダ）、[+] ボタン、フォルダ・.md ファイルの階層（フォルダ先行、名前順）。編集中ノートは背景強調
- **操作**: フォルダクリックで展開/折りたたみ。ノートクリックで開く。右クリックでメニュー（リネーム / 削除）
- **遷移先**: —

### エディタ（ノート編集中）

- **表示内容**: 開いているノートの内容（editor エリアの仕様どおり）
- **操作**: 編集（自動保存）、Ctrl+S（即時保存）
- **遷移先**: 別ノートをクリックすると保存してから切り替える

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| 保管庫外のパスへのアクセス | コマンドがエラーを返し、フロントはコンソールへ記録（UI 変化なし） |
| 読み込み失敗（削除済み等） | ステータス表示「読み込みに失敗しました」。エディタは変更しない |
| 保存失敗 | ステータス表示「保存に失敗しました」。内容は保持し次回変更時に再試行 |
| リネーム先が既に存在 | コマンドがエラーを返し、ステータス表示「同名のファイルがあります」 |
| ダイアログをキャンセル | 何もしない |

## 未対応ケース

- 保管庫の記憶・履歴
- ファイル移動（D&D）、フォルダの作成/リネーム/削除
- 外部変更の検知（開いている間に他アプリで編集された場合、後勝ち）
- .md 以外のファイル表示

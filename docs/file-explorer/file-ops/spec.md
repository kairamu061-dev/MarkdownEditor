# file-explorer/file-ops 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | 新規ノート | ヘッダ [+]。名前は「無題のノート.md」、既存なら「無題のノート 2.md」…と連番。作成後すぐ開く |
| 2 | コンテキストメニュー | ノート行の右クリックで表示。項目: リネーム / 削除。メニュー外クリックか Esc で閉じる |
| 3 | リネーム | 行がテキスト入力欄になる（初期値は拡張子抜きの現名称）。Enter 確定（`.md` を自動付与）、Esc キャンセル |
| 4 | 削除 | メニューの「削除」を押すと項目が「本当に削除する」に変わり、もう一度押すと実行（ごみ箱へ） |

## 画面・操作フロー

1. [+] を押すと「無題のノート.md」が作成され、ツリーに現れ、エディタで開かれる
2. ノート行を右クリックするとカーソル位置にメニューが出る
3. 「リネーム」→ 行が入力欄に変わる → 名前を編集して Enter → ツリーに新名称で反映
4. 「削除」→「本当に削除する」→ ノートがごみ箱へ移動しツリーから消える

## 各画面 / 状態の詳細

### コンテキストメニュー

- **表示内容**: リネーム / 削除 の 2 項目。`--bg-hover` 背景のフローティングボックス
- **操作**: 項目クリック。メニュー外クリック / Esc / 別の右クリックで閉じる
- **遷移先**: リネーム = インライン入力状態、削除 = 確認状態

### インライン入力（リネーム中）

- **表示内容**: 行の位置にテキスト入力欄
- **操作**: Enter = 確定、Esc / フォーカス喪失 = キャンセル
- **遷移先**: 通常のツリー表示

## 編集中ノートへの追随

- リネーム対象が編集中ノートの場合、保存先パスを新パスへ更新する（保存は継続動作）
- 削除対象が編集中ノートの場合、エディタを空にし保存対象なし（スクラッチ扱い）に戻す

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| リネーム先が既存 | ステータス「同名のファイルがあります」。入力状態を維持 |
| 空文字・`/` を含む名前 | 確定せず入力状態を維持 |
| create/delete の失敗 | ステータス「操作に失敗しました」 |

## 未対応ケース

- フォルダ操作、任意フォルダ配下への作成（右クリックからの「ここに作成」）
- F2 / Delete キーでの操作
- 複数選択での一括操作

# file-explorer/file-ops 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | 新規ノート | ヘッダ [+] はルート直下に作成。名前は「無題のノート.md」、既存なら「無題のノート 2.md」…と連番。作成後すぐ開く |
| 2 | コンテキストメニュー | ノート行・フォルダ行・ツリー空白部の右クリックで表示。メニュー外クリックか Esc で閉じる |
| 3 | 新規作成（メニュー） | 「新規ノート」「新規フォルダ」。作成先はフォルダ上ならその中、ノート上なら同じ親、空白部ならルート。フォルダ名は「新規フォルダ」を連番 |
| 4 | リネーム | 行がテキスト入力欄になる（初期値は現名称、ノートは拡張子抜き）。Enter 確定（ノートは `.md` を自動付与）、Esc キャンセル。フォルダも対象 |
| 5 | 削除 | 「削除」を押すと「本当に削除する」（フォルダは「フォルダごと削除する」）に変わり、もう一度押すと実行（ごみ箱へ）。フォルダは中身ごと |

## 画面・操作フロー

1. [+] を押すと「無題のノート.md」が作成され、ツリーに現れ、エディタで開かれる
2. ノート行を右クリックするとカーソル位置にメニューが出る
3. 「リネーム」→ 行が入力欄に変わる → 名前を編集して Enter → ツリーに新名称で反映
4. 「削除」→「本当に削除する」→ ノートがごみ箱へ移動しツリーから消える

## 各画面 / 状態の詳細

### コンテキストメニュー

- **表示内容**: 新規ノート / 新規フォルダ（常に）＋ リネーム / 削除（実項目の上のみ、ルート空白部では出さない）。`--bg-hover` 背景のフローティングボックス
- **操作**: 項目クリック。メニュー外クリック / Esc / 別の右クリックで閉じる
- **遷移先**: 新規作成 = ツリー更新（ノートは作成後に開く）、リネーム = インライン入力状態、削除 = 確認状態

### インライン入力（リネーム中）

- **表示内容**: 行の位置にテキスト入力欄
- **操作**: Enter = 確定、Esc / フォーカス喪失 = キャンセル
- **遷移先**: 通常のツリー表示

## 編集中ノートへの追随

- リネーム対象が編集中ノートの場合、保存先パスを新パスへ更新する（保存は継続動作）
- 削除対象が編集中ノート、またはそれを含むフォルダの場合、エディタを空にし保存対象なし（スクラッチ扱い）に戻す
- フォルダのドラッグ移動に伴う編集中ノートのパス付け替えは tree-view 側で行う

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| リネーム先が既存 | ステータス「同名の項目があります」。入力状態を維持 |
| 空文字・`/` を含む名前 | 確定せず入力状態を維持 |
| create/delete の失敗 | ステータス「操作に失敗しました」 |

## 未対応ケース

- 新規作成後のフォルダ名インライン編集への自動遷移（現状は「新規フォルダ」名で作成、以後手動リネーム）
- F2 / Delete キーでの操作
- 複数選択での一括操作

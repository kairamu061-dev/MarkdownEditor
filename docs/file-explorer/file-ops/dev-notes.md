# file-explorer/file-ops 開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| 削除確認はネイティブダイアログでなくメニュー内の二段階クリック | WebView の confirm() は環境依存で信頼できず、ダイアログプラグインの JS API 追加も不要になる。誤クリック防止には十分 |
| 新規ノートの連番は already exists エラーでリトライ | フロントでツリー状態から計算するより単純で、競合にも強い |
| コンテキストメニューは自前 DOM | ネイティブメニューより Nord テーマとの統一が容易（設計どおり） |

## 発生した問題と対処

| 問題 | 対処 |
|------|------|
| E2E スクリプトの座標指定ミスでメニュー項目を外し、意図しないノートを削除した | アプリ側のバグではない（テスト用保管庫のため実害なし。削除はごみ箱行きで復元可能）。メニュー項目の座標はクリック位置 +20px / +48px と判明したので再検証で修正 |

# file-explorer/file-ops 概要

## 目的・背景

ノートのライフサイクル操作（新規作成・リネーム・削除）を提供する。
vault のコマンドと tree-view の UI 基盤の上に、Obsidian と同様の
「[+] で即作成」「右クリックメニューで操作」の体験を実装する。

## スコープ

### 作るもの

- サイドバーヘッダの [+] ボタン（保管庫直下に「無題のノート N.md」を作成して開く）
- ノート行の右クリックコンテキストメニュー（リネーム / 削除）
- リネーム: 行がインライン入力欄になり、Enter で確定・Esc でキャンセル
- 削除: メニュー内で二段階確認（「削除」→「本当に削除する」）の後、ごみ箱へ
- 操作後のツリー自動更新と、編集中ノートへの追随（リネーム時はパス更新、削除時はスクラッチへ）

### 作らないもの

- フォルダの作成・リネーム・削除
- 任意フォルダ配下への新規作成（v1 は保管庫直下のみ）
- キーボードショートカット（F2 / Delete）

## 制約

- ファイル操作は vault のコマンド（create_note / rename_path / delete_path）のみを使う
- メニュー等の UI 色は nord.css のエイリアスを参照

## 完了条件

- [+] でノートが作成されエディタで開かれる。連打しても重複名にならない
- 右クリックメニューからリネーム・削除ができ、ツリーへ即時反映される
- 編集中ノートをリネームしても保存先が追随する。削除するとエディタは空になる

## 画面イメージ

```
| 保管庫名 [+] |        右クリック時:
| • ノートA        |        +--------------+
| • ノートB ←右クリ |        | リネーム      |
|                  |        | 削除          |
|                  |        +--------------+
```

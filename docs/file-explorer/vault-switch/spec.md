# file-explorer/vault-switch 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | スイッチャ起動 | サイドバー**下部（フッタ）**の保管庫名（＋キャレット ⌃）をクリックでプルダウンを開閉（2026-07-11 フィードバックでヘッダから下部へ変更） |
| 2 | 最近の保管庫一覧 | `recentVaults` を新しい順に表示。各項目はフォルダ名（basename）、ツールチップにフルパス。現在の保管庫は強調 |
| 3 | 保管庫の切り替え | 一覧の項目クリックで `open_vault(path)` を呼び、その保管庫のツリーへ切り替え |
| 4 | 別の保管庫を開く | 「別の保管庫を開く…」で `pick_vault`（フォルダ選択ダイアログ） |
| 5 | 履歴の永続化 | 保管庫を開くたびに `recentVaults` の先頭へ積む（重複除去・最大 10 件）。settings.json に保存 |
| 6 | 履歴からの削除 | 一覧項目の右クリックで「履歴から削除」メニュー。選ぶと `recentVaults` からそのパスを除去して一覧を更新（2026-07-12 追加） |

## 画面・操作フロー

1. サイドバー下部（フッタ）の保管庫名をクリックするとプルダウンが上方向に開く
2. 最近開いた保管庫が新しい順に並び、現在の保管庫はアクセント色で示される
3. 別の保管庫名をクリックすると、その保管庫のツリーに切り替わる（開いていたノートは閉じる）
4. 「別の保管庫を開く…」を選ぶとフォルダ選択ダイアログが開き、選んだフォルダを開く
5. メニュー外クリック / Esc / 再クリックで閉じる

## 各画面 / 状態の詳細

### 保管庫スイッチャ（プルダウン）

- **表示内容**: 最近の保管庫（basename）一覧 → 区切り線 → 「別の保管庫を開く…」。`--bg-hover` 背景のフローティングボックス。フッタの上に上方向へ展開
- **操作**: 項目クリックで切り替え / ダイアログ。現在の保管庫クリックは何もしない。メニュー外クリック・Esc・保管庫名の再クリックで閉じる
- **遷移先**: 切り替え成功時は選択した保管庫のツリー表示

### 履歴項目の右クリックメニュー（2026-07-12 追加）

- **表示条件**: 一覧項目の右クリック。**現在開いている保管庫の項目には出さない**
  （lastVault が指しているため、削除しても次回オープンで復活して紛らわしい）
- **表示内容**: 「履歴から削除」の 1 項目（ツリーのコンテキストメニューと同じ見た目）
- **動作**: `recentVaults` からそのパスを除去して settings.json に保存し、スイッチャの一覧を開いたまま更新。
  **ディスク上の保管庫フォルダには一切触れない**（履歴のみの削除）
- **閉じる**: メニュー外クリック / Esc（スイッチャ本体を閉じるとメニューも閉じる）

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| `open_vault` 失敗（フォルダが存在しない等） | ステータス「保管庫を開けませんでした」。現在の表示は維持 |
| `pick_vault` 失敗 | ステータス「保管庫を開けませんでした」 |
| ダイアログキャンセル | 何もしない |
| `get_settings` 失敗 | 一覧は空として扱い、「別の保管庫を開く…」のみ表示 |

## 未対応ケース

- 存在しなくなった保管庫の一覧からの自動除去（開こうとして失敗した時点でエラー表示のみ。
  2026-07-12 から右クリック → 履歴から削除で手動除去は可能）
- 保管庫のお気に入り・並び替え
- 現在開いている保管庫の履歴からの削除（上記のとおり意図的に対象外）
- キーボードのみでの一覧操作（矢印キーによる選択）

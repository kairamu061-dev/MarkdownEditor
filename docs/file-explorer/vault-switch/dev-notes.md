# file-explorer/vault-switch 開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| recentVaults は settings に相乗りし `remember_vault` の一箇所で更新 | 保管庫を開く経路（pick / open / initial）すべてが remember_vault を通るため、履歴更新を漏れなく一元化できる |
| 一覧の表示名は basename、フルパスはツールチップ | サイドバー幅が狭く、フォルダ名だけで十分識別できる。同名フォルダはツールチップのフルパスで区別 |
| 切り替え時にエディタを空にし collapsedDirs をリセット | 別保管庫のためノート・開閉状態は無意味。混乱を避けるため applyVault でまとめてリセット |
| スイッチャの開閉は mousedown の外側判定 + Esc | コンテキストメニューと同じ様式。保管庫名ボタン自身は除外して再クリックでのトグルを成立させる |

## 発生した問題と対処

| 問題 | 対処 |
|------|------|
| 外側 mousedown で閉じると、保管庫名ボタンの click より先に閉じてしまいトグルが常に開く | 外側判定から `.vault-switcher-button` を除外し、ボタンの click 側でトグルを判定 |

## 設計からの変更点

| 変更内容 | 理由 |
|----------|------|
| スイッチャの配置をサイドバーヘッダから下部フッタ（#sidebar-footer）へ変更し、メニューを上方向展開に変更（2026-07-11） | ユーザーフィードバック「保管庫の名前や切り替えは下に移動したい」。Obsidian の配置とも一致する。spec / design / index.html を更新済み |
| フッタに設定 ⚙ ボタンが同居するため、`renderVaultSwitcher` は footer 全体を消さずスイッチャ部分（`.vault-switcher-button`）のみ差し替える（2026-07-12） | 「設定ボタンは保管庫切り替えの横に」対応で settings/ui が ⚙ をフッタへ追加。footer.textContent="" だと ⚙ が消えるため、スイッチャは先頭へ insert・古いスイッチャのみ remove する方式に変更。あわせてフッタ・スイッチャ領域を拡大 |

# file-explorer/vault-switch 概要

## 目的・背景

複数の保管庫（ローカルフォルダ）を扱うユーザーが、アプリを再起動せずに保管庫を切り替えられるようにする。
Obsidian の保管庫スイッチャに倣い、サイドバーの保管庫名から最近開いた保管庫の一覧を出し、
その場で切り替えられる導線を提供する。

## スコープ

### 作るもの

- サイドバーヘッダの保管庫名をクリックで開くプルダウン（保管庫スイッチャ）
- 最近開いた保管庫の一覧表示（新しい順・最大 10 件）と、選択による切り替え
- 「別の保管庫を開く…」からのフォルダ選択（既存の `pick_vault` を利用）
- 最近開いた保管庫の永続化（settings.json の `recentVaults`）

### 作らないもの

- 保管庫のブックマーク・お気に入り・並び替え UI
- 保管庫一覧からの個別削除
- 複数保管庫の同時表示（タブ / 分割）

## 制約

- 保管庫の実体解決・ツリー構築は既存の vault サブ項目（Rust 側）に委譲する
- 切り替え時は開いていたノートと開閉状態をリセットする（別保管庫のため）

## 完了条件

- 保管庫名クリックで最近の保管庫一覧＋「別の保管庫を開く…」が表示される
- 一覧の項目を選ぶとその保管庫のツリーに切り替わる
- 切り替えた保管庫が recentVaults の先頭に来て、再起動後も一覧に残る

## 画面イメージ

```
+---------------------------+
|  [ 保管庫名        ⌄ ]  ← クリック
+---------------------------+
| ✓ 保管庫名（現在）        |
|   仕事ノート              |
|   日記                    |
| ------------------------- |
|   別の保管庫を開く…       |
+---------------------------+
```

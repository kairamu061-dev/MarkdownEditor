# file-explorer/vault 仕様

## 機能一覧（コマンド）

| # | コマンド | 説明 |
|---|---------|------|
| 1 | `pick_vault` | フォルダ選択ダイアログを開き、選択されたら保管庫として設定。保管庫名と相対パスツリーを返す。キャンセル時は null |
| 2 | `initial_vault` | 起動引数（ディレクトリの場合）または環境変数 `MDE_VAULT` から保管庫を設定して返す。未指定なら null |
| 3 | `list_tree` | 現在の保管庫のツリー（フォルダと .md のみ、フォルダ先行・名前順）を返す |
| 4 | `read_note(rel_path)` | ノート内容を返す |
| 5 | `write_note(rel_path, content)` | ノートを保存する |
| 6 | `create_note(rel_path)` | 空ノートを新規作成する（既存ならエラー） |
| 7 | `rename_path(from, to)` | リネームする（to が既存ならエラー） |
| 8 | `delete_path(rel_path)` | OS のごみ箱へ移動する |

## パス検証ルール（全コマンド共通）

- 受け取る相対パスを保管庫ルートに連結し正規化した結果が、保管庫ルート配下でなければエラー
- `..` を含むパス、絶対パス、保管庫未設定時の呼び出しはエラー
- 対象は `.md` ファイルとディレクトリのみ（read/write/create/rename/delete は `.md` のみ）

## ツリーのデータ仕様

- ノードは `{ name, path（保管庫相対）, isDir, children }`
- 各階層でフォルダ先行、名前昇順（大文字小文字無視）
- 隠しファイル・隠しフォルダ（`.` 始まり）は含めない

## エラーケース

| 条件 | 挙動 |
|------|------|
| 保管庫未設定でのコマンド呼び出し | `"vault is not open"` エラー |
| 保管庫外パス | `"path is outside the vault"` エラー |
| create/rename の対象が既存 | `"already exists"` エラー |
| 対象が .md でない | `"not a markdown file"` エラー |

## 未対応ケース

- シンボリックリンクの解決（保管庫内判定は正規化パスで行い、リンク先の実体は追わない）
- 同時書き込みの排他制御（単一ウィンドウのため不要）

# file-explorer/vault 概要

## 目的・背景

保管庫（開いているフォルダ）の状態管理とファイル I/O を Rust 側に集約する。
tree-view / file-ops が使うすべてのコマンドの提供元であり、
保管庫外パスの拒否（パストラバーサル防止）をここで一元的に担う。

## スコープ

### 作るもの

- 保管庫状態（ルートパス）の管理と設定コマンド
- 起動時保管庫の解決（起動引数 / 環境変数 `MDE_VAULT`）
- フォルダ選択ダイアログ（tauri-plugin-dialog）
- ツリー取得・ノート読み書き・作成・リネーム・ごみ箱削除のコマンド
- パス検証（相対パス解決と保管庫内チェック）のユニットテスト

### 作らないもの

- UI（→ tree-view / file-ops）
- ファイルウォッチャ・検索

## 制約

- コマンド境界のパスは保管庫相対で統一（file-explorer/design.md の横断事項）
- 削除は `trash` クレートで OS のごみ箱へ移動する

## 完了条件

- 全コマンドが実装され、`cargo test` でパス検証のユニットテストが通る
- 保管庫外パス（`../` 等）を含む呼び出しがエラーになる

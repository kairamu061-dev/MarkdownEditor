# file-explorer/vault 開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| コマンド境界は保管庫相対パスで統一 | フロントに絶対パスを渡さないことで、検証を resolve_in_vault の 1 箇所に集約できる |
| resolve_in_vault は `..` を字面で拒否（canonicalize しない） | 未作成パス（create_note）にも同じ検証を使えるため。シンボリックリンク追跡はしない旨を spec に明記 |
| 削除は trash クレートでごみ箱へ | 誤操作からの復旧可能性。Windows でもごみ箱に対応 |
| ファイル操作は .md のみ許可 | エディタが扱えないファイルの破壊を防ぐ |

# file-explorer 概要

## 目的・背景

ヴォールト（ローカルフォルダ）を開き、その中のノートを閲覧・編集・保存できるようにする。
editor エリアと接続することで「フォルダを開く → ノートを選ぶ → 編集する → 保存される」という
エディタとしての基本フローが完成する。

## スコープ

### 作るもの

- ヴォールトを開く（フォルダ選択ダイアログ / 起動引数・環境変数での指定）
- Rust 側のヴォールト管理とファイル I/O コマンド（ツリー取得・読み書き。パスはヴォールト内に制限）
- サイドバーのファイルツリー（フォルダ展開/折りたたみ、ノートを開く、編集中ノートの強調）
- 保存（変更のデバウンス自動保存 + Ctrl+S 即時保存）
- ノートの新規作成・リネーム・削除（コンテキストメニュー）

### 作らないもの

- ヴォールトの記憶（次回起動時の自動再オープン。設定機能の実装時に対応）
- ヴォールト内全文検索（将来の別エリア）
- .md 以外のファイルの表示・編集
- ファイルの移動（ドラッグ&ドロップ）・フォルダの作成/リネーム/削除
- 外部での変更の監視（ファイルウォッチャ）

## 制約

- ファイルアクセスはすべて Rust 側の Tauri コマンド経由（project_overview.md のアーキテクチャ）
- すべてのコマンドは開いているヴォールトの外のパスを拒否する（パストラバーサル防止）
- 削除は完全削除ではなく OS のごみ箱へ移動する

## 完了条件

- フォルダを開くとサイドバーに .md ファイルのツリーが表示される
- ノートをクリックするとエディタに内容が表示され、編集すると自動保存される（Ctrl+S でも保存）
- ノートの新規作成・リネーム・削除ができ、ツリーに反映される
- サブ項目（vault / tree-view / file-ops）の完了条件をすべて満たす

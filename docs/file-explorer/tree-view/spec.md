# file-explorer/tree-view 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | ヴォールトを開く | 未オープン時はボタン表示。押下で `pick_vault`。起動時は `initial_vault` を試行 |
| 2 | ツリー描画 | ヘッダにヴォールト名。フォルダは ▸/▾ で展開状態を表示（初期は全展開）。ノートは名前から `.md` を省いて表示 |
| 3 | ノートを開く | クリックで `read_note` → `setContent`。開いたノートに `active` 強調 |
| 4 | 自動保存 | 変更停止から 700ms 後に `write_note`。ノート切り替え・Ctrl+S 時は即時保存 |
| 5 | ステータス表示 | サイドバー下部に 1 行。エラー時のみ表示（数秒で消える） |

## 画面・操作フロー

親 spec.md の「画面・操作フロー」1〜4 を参照。

## 各画面 / 状態の詳細

### サイドバー（ヴォールト未オープン）

- **表示内容**: 中央に「ヴォールトを開く」ボタン
- **操作**: 押下 → フォルダ選択 → 選択時はツリー表示へ、キャンセル時はそのまま
- **遷移先**: ツリー表示

### サイドバー（ツリー表示）

- **表示内容**: ヘッダ = ヴォールト名。ツリー = vault の返す順序どおり（フォルダ先行・名前順）。編集中ノートは `--bg-hover` 背景 + `--text-strong`
- **操作**: フォルダ行クリック = 展開/折りたたみ。ノート行クリック = 開く
- **遷移先**: —

## 保存の詳細仕様

- 編集のたびにデバウンスタイマー（700ms）をリセットし、満了時に `write_note`
- 以下のタイミングでは満了を待たず即時保存する: Ctrl+S、別ノートへの切り替え直前
- 保存失敗時はステータス表示し、内容は保持（次の変更で再試行）
- ノートを開いていない状態（スクラッチ文書）では保存しない

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| `read_note` 失敗 | ステータス「読み込みに失敗しました」。エディタは変更しない |
| `write_note` 失敗 | ステータス「保存に失敗しました」 |
| ダイアログキャンセル | 何もしない |

## 未対応ケース

- ツリーの手動リフレッシュ（file-ops の操作後は自動更新される）
- 展開状態の記憶

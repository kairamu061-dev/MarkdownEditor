# file-explorer/tree-view 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | 保管庫を開く | 未オープン時はボタン表示。押下で `pick_vault`。起動時は `initial_vault` を試行。保管庫の切り替えは vault-switch サブ項目を参照 |
| 2 | ツリー描画 | フォルダは行**左端**の開閉マーク ▸/▾（視認しやすい大きめ表示）で展開状態を表示。ノートは名前から `.md` を省いて表示。保管庫名の表示は下部フッタ（vault-switch）へ移動（2026-07-11 フィードバック） |
| 3 | 見た目の差別化 | フォルダは強調色＋やや太字で開閉マークあり、ノートは通常色でマークなし |
| 4 | 階層インデント | 子階層はコンパクトにインデント（1 階層あたり約 12px。2026-07-11 フィードバックで従来の半分に縮小）し、ネスト階層に薄いガイド線を表示 |
| 5 | 開閉状態の保持 | フォルダの開閉状態を JS 側（相対パスの集合）で保持し、ノートを開いても再描画で失われない（BUG-002） |
| 6 | ノートを開く | クリックで `read_note` → `setContent`。開いたノートに `active` 強調 |
| 7 | ドラッグ&ドロップ移動 | ノート・フォルダを別フォルダ / ルートへドラッグで移動（`move_path`）。移動先をハイライトし、自身・子孫への移動は不可 |
| 8 | 自動保存 | 変更停止から 700ms 後に `write_note`。ノート切り替え・Ctrl+S 時は即時保存 |
| 9 | ステータス表示 | サイドバー下部に 1 行。エラー時のみ表示（数秒で消える） |

## 画面・操作フロー

親 spec.md の「画面・操作フロー」1〜4 を参照。

## 各画面 / 状態の詳細

### サイドバー（保管庫未オープン）

- **表示内容**: 中央に「保管庫を開く」ボタン
- **操作**: 押下 → フォルダ選択 → 選択時はツリー表示へ、キャンセル時はそのまま
- **遷移先**: ツリー表示

### サイドバー（ツリー表示）

- **表示内容**: ヘッダ = [+] / ⚙ ボタン。下部フッタ = 保管庫名（クリックで vault-switch のスイッチャ）。ツリー = vault の返す順序どおり（フォルダ先行・名前順）。フォルダは強調色＋太字＋開閉マーク、ノートは通常色。編集中ノートは `--bg-hover` 背景 + `--text-strong`
- **操作**: フォルダ行クリック = 展開/折りたたみ。ノート行クリック = 開く。行のドラッグ&ドロップ = 移動。右クリック = コンテキストメニュー（file-ops）
- **遷移先**: —

## ドラッグ&ドロップ移動の詳細仕様

- ノート・フォルダの行をドラッグでき、フォルダ行またはツリー空白部（ルート）をドロップ先にできる
- ドロップ可能な対象にはハイライト（破線アウトライン）を表示する
- 次の移動は不可（ドロップを受け付けない）: 自身、自身の子孫フォルダ、現在の親（同じ場所）
- 移動後は開閉状態（collapsedDirs）と編集中ノートのパスを移動先へ付け替える
- 移動先に同名がある場合はステータス「移動先に同名の項目があります」

## 保存の詳細仕様

- 編集のたびにデバウンスタイマー（700ms）をリセットし、満了時に `write_note`
- 以下のタイミングでは満了を待たず即時保存する: Ctrl+S、別ノートへの切り替え直前
- 保存失敗時はステータス表示し、内容は保持（次の変更で再試行）
- ノートを開いていない状態（スクラッチ文書）では保存しない

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| `read_note` 失敗 | ステータス「読み込みに失敗しました」。エディタは変更しない |
| `write_note` 失敗 | ステータス「保存に失敗しました」 |
| ダイアログキャンセル | 何もしない |

## 未対応ケース

- ツリーの手動リフレッシュ（file-ops の操作後は自動更新される）
- 開閉状態のディスク永続化（セッション内のみ保持。再起動では全展開に戻る）
- キーボードによるツリー操作（矢印キーでの移動・展開）

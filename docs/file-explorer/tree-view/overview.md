# file-explorer/tree-view 概要

## 目的・背景

サイドバーに保管庫のファイルツリーを表示し、ノートを開く・編集する・保存するという
基本フローのフロントエンド側を実装する。vault のコマンドと editor の公開 API を接続する役割を持つ。

## スコープ

### 作るもの

- 保管庫未オープン時の「保管庫を開く」ボタン
- 起動時保管庫（`initial_vault`）の自動オープン
- ファイルツリーの描画（フォルダの展開/折りたたみ、編集中ノートの強調）
- ノートを開く（クリック → `read_note` → エディタへ）
- 保存（`onDocChanged` から 700ms デバウンスで `write_note`、Ctrl+S で即時）
- ステータス表示（保存失敗等のエラー通知欄）

### 作らないもの

- 新規作成・リネーム・削除（→ file-ops）
- ツリーの仮想スクロール（大規模保管庫最適化は将来）

## 制約

- vault のコマンド以外でファイルにアクセスしない
- editor のコードは変更せず、公開 API（`setContent` / `onDocChanged` 等）のみを使う

## 完了条件

- 保管庫を開くとツリーが表示され、ノートのクリックで内容がエディタに載る
- 編集後、自動保存によりディスク上のファイル内容が更新される（Ctrl+S も同様）
- ノート切り替え時に未保存の変更が失われない（切り替え前に保存される）

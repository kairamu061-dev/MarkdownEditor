# editor/live-preview 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | 見出しマーク隠蔽 | `#`〜`######` と直後の空白 1 つを隠す |
| 2 | 強調マーク隠蔽 | `**` `*` `_` `__`（太字・斜体）を隠す |
| 3 | 打ち消しマーク隠蔽 | `~~` を隠す |
| 4 | インラインコードマーク隠蔽 | `` ` `` を隠す（背景色と等幅は core のスタイルで維持） |
| 5 | インラインリンク整形 | `[テキスト](URL)` の `[` `]` `(URL)` を隠しテキストのみ表示 |
| 6 | 弾丸置換 | 箇条書きの `-` `*` `+` をアクセント色の `•` に置換 |

## 表示/再表示の判定ルール

- 判定単位は「記法の構文ノード全体」（例: `**太字**` なら開始マークから終了マークまで）
- 選択範囲（カーソル含む）が構文ノードの範囲と 1 文字でも重なる場合、そのノードの記法マークをすべて表示する
- 重ならない場合は記法マークを隠す
- 複数選択（マルチカーソル）の場合はいずれかの選択が重なれば表示する

## 画面・操作フロー

1. 文書を開くと、カーソル位置以外の記法マークはすべて隠れた整形表示になっている
2. 整形表示された箇所をクリックする（またはカーソル移動で入る）と、その構文ノードの記法マークが再表示される
3. カーソルが構文ノードの外へ出ると、記法マークが再び隠れる

## 各画面 / 状態の詳細

### 整形表示状態（カーソル外）

- **表示内容**: 記法マークなしの整形テキスト。リンクはテキストのみ（下線+アクセント色）、箇条書きは `•`
- **操作**: クリック / カーソル移動で該当ノードがソース表示状態に遷移

### ソース表示状態（カーソル内）

- **表示内容**: 記法マークを含む生テキスト（core のハイライトスタイルで着色）
- **操作**: 通常の編集。カーソルが出ると整形表示状態へ戻る

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| 閉じられていない記法（`**foo` 等） | パーサが強調と認識しないため隠蔽対象外。生テキストのまま表示 |

## 未対応ケース

- 引用マーク `>` の隠蔽と左ボーダー表示（将来対応）
- 画像記法 `![alt](url)` の整形（リンク扱いせずソース表示のまま）
- 番号付きリストの番号スタイル変更
- コードブロックフェンス（```）の隠蔽

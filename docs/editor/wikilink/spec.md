# editor/wikilink 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | リンク表示 | `[[名前]]` の内側をアクセント色+下線で表示。カーソル外では `[[ ]]` を隠す（live-preview と同じ判定ルール） |
| 2 | クリックオープン | 整形表示状態（カーソル外）のリンクをクリックすると該当ノートを開く。カーソル内（ソース表示）でのクリックは通常のカーソル移動 |
| 3 | 名前解決 | `名前.md` をヴォールト全体から大文字小文字無視で検索（フォルダ横断。最初の一致を開く） |
| 4 | 除外 | フェンスコードブロック・インラインコード内の `[[...]]` はリンク化しない |

## 記法

- `[[名前]]`。名前に `[` `]` を含むものは対象外
- エイリアス（`|`）・見出し（`#`）記法は v1 では丸ごとリンク名として扱わない（リンク化しない）

## 画面・操作フロー

1. `[[` と `]]` でノート名を囲むと、カーソルが離れた時点で括弧が隠れリンク表示になる
2. リンクをクリックすると該当ノートが開く（未保存の変更は tree-view の仕様どおり切り替え前に保存）
3. リンク部分にカーソルを入れると `[[名前]]` のソース表示に戻り編集できる

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| 該当ノートが見つからない | ステータス表示「ノートが見つかりません: {名前}」 |
| ヴォールト未オープン | リンク表示はされるが、クリック時にステータス表示（見つからない扱い） |

## 未対応ケース

- `[[名前\|表示名]]`・`[[名前#見出し]]`（リンク化しない）
- 入力補完・存在チェックによる色分け（未解決リンクの別色表示）
- リンク先の自動作成

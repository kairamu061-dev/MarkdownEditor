# editor 仕様

## 機能一覧

| # | 機能名 | 説明 | サブ項目 |
|---|--------|------|---------|
| 1 | Markdown 編集 | メイン領域での CodeMirror 6 による Markdown テキスト編集 | core |
| 2 | シンタックスハイライト | 見出し・強調・コード・引用・リスト・リンク等の Nord 配色ハイライト | core |
| 3 | エディタ公開 API | 内容の取得・差し替え・変更通知（file-explorer との接続点） | core |
| 4 | ライブプレビュー | カーソル位置に応じた記法マークの表示/非表示 | live-preview |

## 画面・操作フロー

1. アプリを起動するとメイン領域にスクラッチ文書が開いた状態のエディタが表示される
2. ユーザーは通常のテキストエディタと同様に入力・編集する（undo/redo は Ctrl+Z / Ctrl+Y）
3. Markdown 記法を入力すると即座に Nord 配色でハイライトされる
4. カーソルが記法範囲の外に出ると、記法マーク（`**` や `#` 等）が隠れて整形表示になる
5. 整形表示された箇所にカーソルを移動する（クリックまたは矢印キー）と記法マークが再表示される

## 各画面 / 状態の詳細

### エディタ（メイン領域）

- **表示内容**: 開いている文書の全文。行の折り返しあり。最大幅 800px 中央寄せ（Obsidian のリーダブルライン幅に相当）
- **操作**: テキスト入力、選択、コピー/ペースト、undo/redo（CodeMirror 標準キーマップ）
- **遷移先**: なし（単一状態。モード切り替えは live-preview 側の仕様を参照）

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| 不正な Markdown 記法 | エラーにせず、プレーンテキストとしてそのまま表示する（Markdown にエラーは存在しない） |

## 未対応ケース

- ソースモードへの明示的な切り替え UI（将来対応。当面はカーソル移動で記法マークが見える）
- 保存・ファイル接続（file-explorer エリアで対応）
- 表・数式・脚注のハイライト強化（GFM 基本記法のみ対応）

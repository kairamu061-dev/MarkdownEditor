# app-shell 概要

## 目的・背景

すべての機能（エディタ、ファイルツリー等）の土台となる Tauri 2 アプリケーションの骨格を構築する。
ウィンドウ、基本レイアウト、Nord テーマ基盤がなければ他のフィーチャーエリアは実装・検証できないため、最初に着手する。

## スコープ

### 作るもの

- Tauri 2 + TypeScript + Vite のプロジェクトスキャフォールド
- 単一メインウィンドウ（タイトル、初期サイズ、最小サイズ）
- 2 ペイン基本レイアウト（左: サイドバー領域、右: メインコンテンツ領域）
- サイドバーの開閉トグル（ボタンおよび `Ctrl+B`）
- Nord カラーパレットの CSS カスタムプロパティ定義（テーマ基盤）
- Windows 向けビルド設定（MSI / NSIS ターゲット）

### 作らないもの

- エディタ本体（→ 別フィーチャーエリア `editor`）
- ファイルツリーの中身（→ 別フィーチャーエリア `file-explorer`）
- 設定画面・設定の永続化
- ライト/ダークテーマ切り替え（Nord ダーク固定で開始）

## 制約

- 開発環境は Linux devcontainer。Linux 上でのビルド・動作確認まで行い、Windows 実機での最終確認はユーザーに依頼する
- 省メモリ方針のため UI フレームワーク（React 等)は使用せず、素の TypeScript + DOM で実装する
- パレット色は `docs/project_overview.md` の「カラーパレット」を唯一の定義元とする

## 完了条件

- `npm run tauri dev` でウィンドウが起動し、Nord 配色の 2 ペインレイアウトが表示される
- サイドバーがボタンと `Ctrl+B` で開閉できる
- `npm run tauri build` が通る（Linux 上）

# app-shell 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | メインウィンドウ | タイトル「MarkdownEditor」、初期 1200x800、最小 640x480 の単一ウィンドウ |
| 2 | 2 ペインレイアウト | 左サイドバー（初期幅 260px）+ 右メインコンテンツの分割表示 |
| 3 | サイドバー開閉 | トグルボタンと `Ctrl+B` でサイドバーの表示/非表示を切り替え |
| 4 | Nord テーマ基盤 | CSS カスタムプロパティで Nord パレットを定義し全 UI に適用 |

## 画面・操作フロー

1. ユーザーがアプリを起動する
2. Nord 配色のメインウィンドウが表示される（サイドバー: 表示状態）
3. サイドバー上部のトグルボタン、または `Ctrl+B` でサイドバーを閉じる
4. 再度トグルするとサイドバーが開く

## 各画面 / 状態の詳細

### メインウィンドウ（サイドバー表示）

- **表示内容**: 左にサイドバー（nord1 背景、幅 260px）、右にメインコンテンツ（nord0 背景）。サイドバー上端にトグルボタン
- **操作**: トグルボタンクリック / `Ctrl+B`
- **遷移先**: サイドバー非表示状態

### メインウィンドウ（サイドバー非表示）

- **表示内容**: メインコンテンツが全幅表示。トグルボタンはウィンドウ左上に残る
- **操作**: トグルボタンクリック / `Ctrl+B`
- **遷移先**: サイドバー表示状態

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| WebView2 未インストール（Windows） | Tauri 標準のインストール誘導に従う（アプリ側の追加対応なし） |

## 未対応ケース

- サイドバー幅のドラッグリサイズ（file-explorer 実装時に対応）
- サイドバー開閉状態の永続化（設定機能の実装時に対応）
- 多重ウィンドウ
